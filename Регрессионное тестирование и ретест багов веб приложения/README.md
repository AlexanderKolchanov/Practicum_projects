# Регрессионное тестирование и ретест багов веб‑приложения

**Цель:**  
Провести регрессионное тестирование и ретест багов по готовым тест‑кейсам.

**Что сделал:**  
- Провел регрессионное тестирование функционала по утверждённым тест‑кейсам.  
- Перепроверил исправленные баги (ретест) на актуальном тестовом стенде.  
- Зафиксировал результаты в отчётной таблице.  


**Объект тестирования:**  
Тестовый стенд веб‑приложения (версия 1.2.3).


**Используемые инструменты:**  
- Google Документы (чек‑листы, отчёты).  
- Google Таблицы (учёт багов, метрики).
