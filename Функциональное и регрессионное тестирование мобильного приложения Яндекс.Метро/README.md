# Функциональное и регрессионное тестирование мобильного приложения Яндекс.Метро
## Цель
Проверить:
- функциональность, затронутую рефакторингом;
- регрессионные сценарии для мобильного приложения;
- стабильность работы в типовых мобильных условиях.


## Выполнено
1. Подготовлен чек‑лист для **функционального тестирования** (по изменённым требованиям).
2. Составлен чек‑лист для **регрессионного тестирования** с учётом:
   - мобильных сценариев (переключение сетей, геолокация, авиарежим и др.);
   - общих проверок для мобильных приложений (обновление, смена ориентации и т. п.).
3. Проведено тестирование на **эмуляторе Android Studio** (конфигурация: Honor 8, Android 9.0, 1080×1920).
4. Заведены баг‑репорты в Google Таблицу.

## Объект тестирования
- Функциональность Яндекс Метро, затронутая рефакторингом.
- Мобильные особенности:
  - работа с сетями (3G/LTE/Wi‑Fi);
  - геолокация;
  - поведение при уведомлениях/звонках;
  - смена ориентации экрана и масштабирование;
  - режим энергосбережения и низкий заряд.


## Инструменты
- **Android Studio** (эмулятор);
- **Google Таблицы** (чек‑листы, баг‑репорты);
- **Требования к приложению** (для проверки функциональности).


## Результаты
- Проверено **30+ сценариев** (функциональные + регрессионные).
- Выявлено **8 дефектов** (5 — функциональность, 3 — мобильные сценарии).
- Все баги задокументированы с приоритетом и шагами воспроизведения.
